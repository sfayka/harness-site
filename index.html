Trust AI execution less. Verify completion more.

Harness evaluates whether work is actually done instead of trusting agent completion claims on their own.

It is built for AI-assisted workflows where execution, evidence, reconciliation, and review all matter. The goal is not more agent optimism. The goal is canonical truth.

The problem

Execution is not completion.

Agents can say work is done. Logs can say a command succeeded. Neither is enough when teams need durable truth, trustworthy artifacts, and explicit review paths.

The position

Harness treats completion as a governed decision.

It evaluates task state against evidence, reconciliation facts, and review decisions before allowing work to count as complete.

What Harness does

Product framing without product theater.

Canonical task evaluation

Accepts task envelopes and reasons over current state instead of relying on agent-reported success.

Evidence-backed completion

Uses artifacts, facts, and explicit lifecycle transitions to determine whether completion should be trusted.

Governed reconciliation

Repairs recoverable mismatches, then returns to reevaluation rather than silently accepting incomplete work.

Explicit review paths

Escalates when automation should stop and a human decision is required.

Key design rule

Harness spends automation before operator attention.

Why that matters

Teams get clearer truth without pretending every recovery path is safe.

Governed reconciliation and proof

Show the failure path and the success path.

KNO-174

Recovery blocked

Missing PR after execution leads to reconciliation, recovery is blocked, and the task escalates explicitly into review.

Outcome: in_review, not silent success.

KNO-175

Recovery succeeds

Harness repairs the missing artifact, attaches the evidence, reevaluates, and only then allows the task to complete.

Outcome: completed through reevaluation.

This site is intentionally proof-oriented. Harness should be judged by what it actually governs, not by broad AI workflow promises.

Philosophy

Verified beats asserted.

Harness is opinionated about the difference between running work and finishing work. It is built around evidence, lifecycle enforcement, and reviewable decisions because those are the parts that keep automation honest.

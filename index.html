Why it exists

Execution is not completion.

Agents can report success. Commands can finish cleanly. A branch can contain code. None of that is enough on its own when the real question is whether the task is complete and trustworthy.

Harness is built for that distinction. It treats completion as a governed decision, not as an assertion.

What Harness does

Evaluates canonical task state

Harness reasons over the task itself instead of accepting agent output as truth.

Requires evidence for completion

Completion depends on artifacts, coherent run context, and the rules that govern the task lifecycle.

Reconciles recoverable defects

When something is fixable, Harness can repair it and send the task back through reevaluation.

Escalates when review should take over

When automation should stop, Harness makes that explicit.

How it works

01

Ingress

Task envelopes arrive from manual, Linear, or executor-adjacent inputs.

02

Completion claim

Claims are treated as inputs to evaluate, not as completion truth.

03

Evaluation and reconciliation

Harness checks evidence, run context, and recovery conditions.

04

Decision

The task completes, fails, reconciles, or escalates to review.

Design rule: spend automation before operator attention.

Proof

KNO-174

Recovery blocked

Missing PR after execution leads to reconciliation, recovery is blocked, and the task moves into review instead of being treated as complete.

Outcome: in review, not silent success.

KNO-175

Recovery succeeds

Harness repairs the missing artifact, attaches evidence, reevaluates, and only then allows the task to complete.

Outcome: completed through reevaluation.

Current scope

What exists now

  • Python backend for canonical evaluation
  • Read-only Next.js dashboard
  • File and Postgres persistence
  • Completion-claim interception
  • Governed reconciliation for missing PR after execution

What this site does not claim

  • That every reconciliation class is automated
  • That the dashboard is a full mutation surface
  • That Harness is a generic chatbot or PM tool
  • That all integrations are already mature sync systems
